# v0.5.0

### In-app updates
- The app now checks for updates on startup and shows a button in the sidebar when a new version is available. Click to download and install without leaving the app.
- Manual "Check for Updates" button in the Tools section for on-demand checks.
- Uses Tauri's built-in updater with signed update artifacts — the app verifies the signature before applying anything.

### Scan performance
- **Deferred fingerprinting** — file discovery (phase 1) no longer reads file content. SHA-256 fingerprinting is deferred to phase 2, making discovery metadata-only (mtime + size check) for unchanged files.
- **Parallel thumbnail + classification** — thumbnail generation and EXIF extraction now run on a separate thread concurrent with LLM classification, hiding ~200ms per image and ~600ms per PDF behind the GPU-bound Ollama call.
- **Checkpoint resume** — interrupted scans resume from the last processed file instead of restarting phase 2 from the beginning.
- **Batch marker flush** — unchanged file markers are flushed in a single batch before classification starts, avoiding per-file DB round-trips.
- **WalkDir metadata reuse** — discovery uses `entry.metadata()` from the walk instead of separate stat syscalls, halving system calls during the file walk.
- **ETA calculation** — the progress bar now uses per-file timing to estimate remaining time accurately.

### Scan progress fix
- The progress bar and file counter during phase 2 (classification) now update after every file instead of batching every 50. With LLM classification taking a few seconds per file, the old batching made progress look frozen for minutes.

### UI
- The Tools section (Find Duplicates, PDF Passwords, Check for Updates) is now pinned above the Info panel in the sidebar. Content lists (roots, albums, smart folders) scroll independently underneath.

### CI
- Release notes are now loaded from `releases/vX.Y.Z.md` and included in GitHub Releases automatically.
