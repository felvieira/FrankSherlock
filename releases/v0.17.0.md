# Frank Sherlock v0.17.0 — Color Search, GPS Map & Smart Alerts

## What's New

### Color Swatches in Toolbar
- **12-color palette** — a row of named color swatches (Red, Pink, Purple, Indigo, Blue, Cyan, Teal, Green, Yellow, Orange, Brown, Black) appears below the search bar
- **One-click color filter** — clicking a swatch appends `color:#RRGGBB` to the query; clicking the active swatch removes it
- **Clear button** — an ✕ button appears beside the swatches when a color filter is active

### GPS / Map View
- **Map tool** — a new "Map" button in the sidebar Tools section opens an interactive OpenStreetMap view of all photos with GPS coordinates
- **Photo pins** — each GPS-tagged photo appears as a clickable pin; click to see a thumbnail popup with a "Find nearby" button
- **Fit to markers** — the map auto-fits its bounds to show all GPS photos at launch
- **Select on map** — click pins to select them, then use the "Show N selected" button to return to the grid with those files highlighted
- **Context menu** — right-clicking a GPS-tagged photo shows a "Find nearby" option that opens the map

### Saved-Search Alerts
- **Background polling** — every 15 minutes Frank Sherlock checks saved searches that have the 🔔 bell enabled for new matches since the last check
- **Desktop notifications** — when new matches are found, a system notification is fired (requires Notification permission, requested once on first run)
- **In-app toast** — a toast also appears in the app if the window is already open

## Changes

### Backend
- **Migration 22** — adds `gps_lat REAL` and `gps_lon REAL` columns to the `files` table with a partial index on non-NULL GPS rows
- **GPS extraction** — the scan pipeline now persists `ExifLocation.latitude`/`longitude` into `gps_lat`/`gps_lon` on every file upsert
- New Tauri commands: `list_gps_files_cmd`, `find_nearby_cmd`, `check_saved_search_alerts_cmd`
- `find_nearby` uses a ±10° bounding-box pre-filter then sorts by Euclidean distance (up to 50 results by default)
- New models: `GpsFile`, `NearbyResult`, `SavedSearchAlert`

### Frontend
- `Toolbar`: `COLOR_SWATCHES` array; `getActiveColor` / `setColorInQuery` helpers; `.toolbar-color-row` swatch UI
- `MapView`: new component using `maplibre-gl` with OSM raster tiles (dynamic import); pin markers with thumbnail popups
- `useSavedSearchAlerts`: new hook — polls `checkSavedSearchAlerts` on a 15-min interval; requests `Notification` permission on mount
- `App`: `mapMode` state; `enterMapMode()`; `contextMenuHasGps` flag set via `getFileProperties` on right-click; all mode-switch fns exit map mode
- `Sidebar`: `onOpenMap` prop + "Map" button in Tools section
- `ContextMenu`: `hasGps` + `onFindNearby` props; "Find nearby" item shown for GPS files
- `api.ts` / `types.ts`: `GpsFile`, `NearbyResult`, `SavedSearchAlert` types and API functions
- `HelpModal`: documented `color:`, `shot:`, and `blur:` search tokens
- `maplibre-gl ^4.7.1` added to dependencies

### UI
- **Sidebar collapse bug fixed** — pressing the ☰ menu button no longer makes the content area go blank. Root cause: `display:none` on the sidebar removed it from CSS Grid auto-placement, pushing the content into the 0px first column. Fixed by switching `main-area` to Flexbox with an animated width transition on the sidebar instead
- **View mount animation** — switching between modes (Content ↔ Faces ↔ Duplicates ↔ Map etc.) now plays a subtle fade + slide-up entrance animation
