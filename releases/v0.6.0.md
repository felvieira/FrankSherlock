# v0.6.0

### Video support
- Scan and classify video files: MP4, MKV, AVI, MOV, WebM, WMV, FLV, M4V, TS, MPG, MPEG.
- Videos are classified via keyframe extraction and LLM analysis — black frames are automatically skipped.
- Thumbnails are generated from representative keyframes. A play icon badge appears on video tiles.
- Embedded and external subtitles (.srt) are parsed and indexed for full-text search.
- Video metadata (duration, resolution, codecs) is shown in the Properties panel.
- Preview modal plays videos inline via a local streaming server with HTTP Range support.
- Search recognizes `video`, `movie`, `episode`, `film`, and `tv show` as media type keywords.
- Requires ffmpeg/ffprobe (optional) — the setup screen shows availability status.

### Three-phase scanning
- The scan pipeline is now discovering → thumbnailing → classifying → cleanup (was two phases).
- Files appear in the grid immediately with thumbnails while LLM classification runs in the background. Unclassified files show with confidence 0 and extension-inferred media types.
- Thumbnailing and classifying phases each support independent cancel and resume.
- Phase-specific progress: "Thumbnailing X/Y" with add/mod/moved stats, "Classifying X/Y" with ETA.

### Refresh Metadata (no Ollama required)
- Right-click a folder and choose "Refresh Metadata" to discover new/modified files and generate thumbnails without running LLM classification. Works even when Ollama is not running.
- "Rescan" remains the full pipeline including classification.

### Directory tree browser
- Clicking a root folder in the sidebar reveals a collapsible directory tree below it.
- Click any subdirectory to filter the grid to that path. Click again to deselect.
- Children are lazy-loaded one level at a time from the database.
- Clicking a subdirectory fills `subdir:path` visibly in the search input, so you can further refine the query or save it as a smart folder.

### Search improvements
- FTS5 porter stemmer: searching for "running" now matches "run", "runs", "runner", etc.
- LIKE substring fallback: when FTS returns no results, a case-insensitive substring search is attempted automatically.
- Media-type trigger words are stripped from the FTS query to avoid false positives.
- Fixed placeholder text advertising non-existent syntax.
- Help modal documents `album:`, `subdir:` prefixes and the stemming/fallback behavior.
